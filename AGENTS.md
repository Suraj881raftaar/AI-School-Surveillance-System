# AI School Surveillance System - Agent Instructions

## Project Goal

Develop a professional AI-powered School Surveillance System using:

- Python
- Tkinter
- SQLite
- OpenCV
- Pillow

The project should be production-quality and suitable for a professional GitHub portfolio.

---

# General Rules

- Never rewrite the entire project.
- Never rename files unless absolutely necessary.
- Preserve all working functionality.
- Maintain backward compatibility.
- Do not modify unrelated modules.
- Always explain why a change is required.

---

# Coding Standards

- Follow PEP8.
- Use type hints.
- Write docstrings.
- Keep functions small.
- Avoid duplicate code.
- Follow SOLID principles where practical.

---

# Before Editing Code

Always:

1. Read related modules.
2. Understand dependencies.
3. Explain the problem.
4. Explain the solution.
5. Modify the minimum number of files.

---

# Camera Rules

- Never freeze the Tkinter UI.
- Never use blocking loops.
- Always release the camera.
- Always release OpenCV resources.
- Handle camera unavailable errors.

---

# Database Rules

- Use parameterized SQL.
- Always close database connections.
- Use try/finally where appropriate.

---

# Git Workflow

One feature per commit.

Commit message format:

Phase X: Feature Name

Example:

Phase 2: Register Face improvements

---

# Development Workflow

Analyze

↓

Explain

↓

Modify

↓

Test

↓

Commit

Never skip the analysis phase.
